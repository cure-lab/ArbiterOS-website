功能优势

不是只转发模型，而是治理整个 Agent 执行链路

策略防护 + 用户确认

当策略触发拦截时，内核会返回确认信息并等待用户 Yes/No 决策，避免高风险工具调用直接执行。

指令级解析与结构化落盘

将结构化输出、工具调用、工具结果统一映射为指令流，并按 trace 生成可回放文件，支持审计与复盘。

全链路可观测

同时记录 pre-call/post-call 日志，接入 Langfuse 追踪与节点持久化，便于监控性能、行为和异常。

技能信任扫描

可选接入 skill scanner，对技能包进行信任度评估并缓存结果，降低不可信技能带来的执行风险。

OpenClaw 开箱集成

对外暴露 OpenAI 兼容接口，配置后可作为 OpenClaw 默认模型提供者，平滑接入现有工作流。

一键安装与弹性部署

支持安装脚本、用户级 systemd、Docker Compose 与本地源码编译，覆盖个人与团队环境。

快速上手

三步将 ArbiterOS 接入你的 Agent 体系

1

安装并启动 Kernel

运行安装脚本或 Docker Compose，启动 ArbiterOS Kernel（默认 `:4000`）。

2

配置模型与策略

在 `litellm_config.yaml` 配置模型、API Key、可选技能信任扫描参数和策略规则。

3

连接 OpenClaw / 你的 Agent

将模型 Provider 指向 `http://127.0.0.1:4000/v1`，即刻获得治理、审计和可观测能力。

使用场景

从 PoC 到生产，ArbiterOS 帮你守住关键边界

高风险工具执行保护

当 Agent 生成潜在危险命令或路径访问时，策略层可拦截、改写并要求人工确认。

合规与审计复盘

通过 trace 文件与指令历史，定位“模型说了什么、调用了什么、为什么被拦截”。

多模型统一入口

用统一代理层管理多家模型供应商，兼顾成本、稳定性与迁移灵活性。

可观测驱动优化

基于 Langfuse 与本地日志追踪响应链路，持续优化 Prompt、策略和工具编排。
